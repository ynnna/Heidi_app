Blocking and downshift unblocking

These simulations accompany the ones summarrised in Honey, Dwyer and Iliescu (2020a; pp 843-844; Figure 8). One of the primary motivations for the Rescorla-Wagner model was seeking to account for the way in which training cues in combination impacted on learning. Take the case of blocking: first A → US and then AB → US (see Table 1), where little learning is apparent to stimulus B . Rescorla-Wagner is able to explain this absence of learning about B. The use of a pooled error term for the change in associative strength for a compound stimulus AB leads to no learning about no learning about B in Phase 2, given A had reached asymptote by the end of Phase 1. In addition, increasing the number of USs between phase 1 (e.g., A → US1) and phase 2 (AB → US1 → US2) results in responding to B that is consistent with the model, as the increase in the number of USs in Phase 2 means A no longer perfectly predicts the outcome and so the pooled error term is positive which allows B to gain associative strength. The problematic finding, for the Rescorla-Wagner model and others, is that that decreasing the number of USs (downshift unblocking; i.e., A → US1 → US2 and then AB → US1) can also lead to learning about B.

Unblocking (learning and responding to B) by a reduction in the USs, has typically been taken as evidence for attentional changes (Pearce & Mackintosh, 2010). However, a speculative account for this unblocking effect can be derived from HeiDI that does not require an assumption of attentional changes. The basis for this speculative account is that the removal of US2 allows a within-compound B - A association to form more effectively during downshift unblocking than during standard blocking. This happens because the US2 - A association is no longer activated and thus the combined error term for A as the target of associations is positive. In turn, this B - A association allows A to borrow the associative properties of A . The design of the simulated data is summarrised in Table 1.

Blocking A → US1 → US2 AB → US1 → US2
Unblocking A → US1 → US2 AB → US1

Functionality

Simulations tab

HeiDI's predictions can be explored using the four sliders.

Sliders

  • Number of trials (range 1 - 100) - Sets the number of trials desired. The slider only allows even numbers, as the simulations assume equal number of A+ and AB- trials.
  • α A (range 0.1 - 1) - Sets the perceived CS value for stimulus A.
  • α B (range 0.1 - 1) - Sets the perceived CS value for the stimulus B.
  • β 1 (range 0.1 - 1) - Sets the perceived value for the US1.
  • β 2 (range 0.1 - 1) - Sets the perceived value for the US2. Note that US2 is set at 0 for Phase 2 for the (downshift) unblocking group.

Buttons

  • button - Link to code of a standalone version, in R, on github.
  • button - downloads the csv file generated in the simulation. It corresponds to the same data that be visualised in the Data tab

Data tab

Every time sliders are adjusted, the simulation dataframe is re-created. This tab shows the data plotted in the figures.
