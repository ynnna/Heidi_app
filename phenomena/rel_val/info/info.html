Relative validity effect

These simulations accompany the ones summarrised in Honey, Dwyer and Iliescu (2020b:pp 171-174; Figure 2). The relative validity effect was one of the phenomena that motivated the impetus for the adoption of a pooled error term. The relative validity design introduced by Wagner, Logan, Haberlandt and Price (1968) is shown in Table 1. The target stimulus (A) that was paired with a reinforcer on 50% of trials (indicated by “+”) and was nonreinforced on the remaining trials (indicated by “-”). Critically, in the true discrimination condition, when A was reinforced it was paired with stimulus B (on AB+ trials), and when A was nonreinforced it was paired with stimulus C (on AC- trials). In contrast, for animals in the pseudo discrimination condition, the AB and AC stimulus pairs were equally often reinforced and nonreinforced (AB+/– and AC+/–). That is, the reinforcement history of A, considered alone, was the same in the two conditions, but the reinforcement history of B and C differed between the true discrimination and pseudo discrimination conditions.

At test, A, B and C were presented individually. Reflecting the fact that in the true discrimination condition B was always reinforced and C never reinforced, conditioned responding to B was higher than to C. In contrast, in the pseudo discrimination condition B and C elicited equivalent levels of responses, reflecting the fact that these stimuli were paired with reinforcement and nonreinforcement equally often. The critical result was that less responding was evident to A in the true discrimination condition than the pseudo discrimination condition.

If the β value is the same for reinforced and nonreinforced trials, then the Rescorla and Wagner (1972) model is constrained to predict that A will have equivalent associative strength in the true discrimination and pseudo discrimination conditions. This forced Rescorla and Wagner (1972) to assume that the β value is higher for reinforced than nonreinforced trials so that the gain in associative strength generated by reinforcement is greater than the loss in associative strength based on nonreinforcement.

In contrast, according to HeiDI, it is possible for A to “borrow” the associative properties of B and C (via A-B and A-C associative links), and furthermore that the development of these A - B and A - C associative links are differentially affected by the reciprocal US - CS links in the true discrimination and pseudo discrimination conditions (even when the A-US associations are equivalent across conditions).

Panel B shows that the values returned for the A - B association for the true discrimination are smaller than those for the A - C association. This is because the US - B association restricts the growth in the A-B association on AB+  trials (because the US and A “compete” as associates of A), while there the absence of the US on AC- trials, means that the A - C association not be affected in the same way. Therefore, A will be able to borrow relatively little of the strong excitatory properties of B, but rather more of the weak inhibitory properties of C.

In contrast, inspection of Panel D shows that the values returned for the A-B and A-C associations are equivalently strong after a pseudo discrimination (the US will interfere with both A-B and A-C links on AB+ and AC+ trials respectively, but not on AB- and AC- trials). And, because B and C are both moderately excitatory, then the presentation of A will be able to borrow excitatory strength from both B and C.

True discrimination AB+ AC- A, B, C

Functionality

Simulations tab

HeiDI's predictions can be explored using the four sliders.

Sliders

  • Number of trials (range 4 - 100) - Sets the number of trials desired. Note: There are four type of trials which constrain the range of possible trial number.
  • α A (range 0.1 - 1) - Sets the perceived CS value for stimulus A.
  • α B (range 0.1 - 1) - Sets the perceived CS value for the stimulus B.
  • α C (range 0.1 - 1) - Sets the perceived CS value for the stimulus C.
  • β (range 0.1 - 1) - Sets the perceived value for the US1.

Buttons

  • button - Link to code of a standalone version, in R, on github.
  • button - downloads the csv file generated in the simulation. It corresponds to the same data that be visualised in the Data tab

Data tab

Every time sliders are adjusted, the simulation dataframe is re-created. This tab shows the data plotted in the figures.
